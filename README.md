# CS-250-SDLC
SNHU CS 250 Software Development Life Cycle


How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?

I interpret user needs by first having clear communication with them and asking a lot of clarifying questions to make sure I understand what they really need. If possible, I would prefer to hold a focus group in person. Creating user stories helps because the specific format tells you exactly what you are looking for. It helps define who the user is, what they need, and why they need it. This makes it easier to turn their ideas into tasks for the development team.
